# Contributing to Wild Garden

Thank you for considering a contribution. Wild Garden is in design and specification phase — we don't have a deployable seed kit yet, but the architecture, governance, and patterns are being shaped now. **Early contributions shape the project deeply.**

## Current Phase

This repository contains the **design specification** for Wild Garden, a cooperative accountability network built on Cloudflare. The implementation hasn't started yet. That means contributions today are about:

- **Architecture and design** — how the seed kit should work
- **Governance** — how the cooperative should be structured (charter, membership, decision-making)
- **Patterns and primitives** — proven Cloudflare patterns we should adopt or adapt
- **Use cases** — descriptions of how this would help your specific institution
- **Constructive critique** — what's wrong, missing, or naive in the spec

We are NOT yet accepting:
- Implementation PRs (no code to contribute against yet)
- Speculative feature requests without use cases

## How to Contribute

### 1. Open an Issue

Issues are the entry point. Use the appropriate template:

- **Architecture proposal** — propose a primitive, pattern, or design choice. Include working code examples where possible (see [Issue #1](https://github.com/NorthwoodsSentinel/wild-garden/issues/1) for the gold standard).
- **Use case** — describe how Wild Garden would help your institution, what's missing for it to work for you.
- **Critique** — flag a gap, error, or naive assumption in the spec.
- **Charter inquiry** — express interest in being part of the founding 12 charter members.

### 2. Discuss Before Building

Because we don't have a seed kit yet, **don't write implementation code without an issue conversation first.** This isn't gatekeeping — it's avoiding wasted work. The architecture is still in flux.

### 3. Pull Requests

When implementation begins (post-charter formation), PRs will be welcome. Until then:
- README, governance docs, architecture docs, and pattern documentation PRs are welcome anytime
- Test data, example configs, and reference materials are welcome anytime
- Implementation code: hold until the seed kit phase opens

### 4. Tone and Conduct

See [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md). The short version: assume good intent, write the way you'd want to be written to, and remember that this project exists for institutions that have been left behind by the security industry. Snark and superiority undermine that mission.

## What Makes a Great Contribution

We took notes from [Issue #1](https://github.com/NorthwoodsSentinel/wild-garden/issues/1):

1. **Specific use case** — name what real-world problem it solves
2. **Working examples** — actual `curl`, `wrangler`, code snippets where possible
3. **Self-contained** — a reader shouldn't need to ask three questions to understand what you're proposing
4. **Humble framing** — "I noticed X, here's what I'd consider" lands better than "this is wrong"
5. **References** — link Cloudflare docs, related repos, prior art

## Ownership and Recognition

Every meaningful contribution is recognized in [CONTRIBUTORS.md](./CONTRIBUTORS.md). The principle is automatic: if you helped, you're listed. No politics, no gatekeeping.

Significant contributors may be invited to join the founding 12 charter members as the cohort forms. Reach out if that interests you.
